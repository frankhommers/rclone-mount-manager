# Rclone Mount Manager — Roadmap

**Date:** 2026-02-21
**Status:** Approved

## Current State

- Avalonia 11.3 / .NET 10 desktop app with Core/GUI/Tests split
- 8 typed option controls (Toggle, ComboBox, EditableComboBox, Numeric, Duration, SizeSuffix, StringList, Text)
- 6 option groups (Mount, VFS Cache, NFS, Filters, General, Remote Control)
- Pin/unpin options, script generation, backend configuration
- RC auto-populate for new profiles
- 110 tests, 0 warnings
- No distribution, no governance docs, no README

## Phase 1: Bugs & Stability (High Priority)

### 1.1 EditableComboBox Visual Verification

**Problem:** Code fix committed (SelectedEnumValue sync in InitializeTypedValues and OnValueChanged), but not visually verified. Diagnostic Serilog logging added to OptionTemplateSelectorFactory.

**Action:** Run the app, select filefabric backend, verify the `url` field renders as an AutoCompleteBox with suggestion dropdown. Check Serilog output for any "Template not found" warnings.

### 1.2 Remove Profile Fix + Empty State

**Problem:** "Remove doesn't work" — minimum 1 profile constraint is enforced but not communicated to the user.

**Action:** Allow deletion of all profiles. Show an empty state screen when no profiles exist ("Create your first profile" with a button). Remove the minimum-1-profile constraint.

### 1.3 ComboBox Dropdown Overflow

**Problem:** Long values (e.g., access_scopes) may overflow the ComboBox dropdown.

**Action:** Ensure all ComboBox ItemTemplates use MaxWidth + TextTrimming + ToolTip for long values.

## Phase 2: Core Features (High Priority)

### 2.1 Mount Status Monitoring via RC API

**Problem:** No way to see which mounts are currently active.

**Action:** Query `mount/listmounts` via the RC API at the `rc_addr` configured per profile. Show a running/stopped indicator per profile in the sidebar. Poll periodically or on demand.

**Depends on:** RC group (done). Profiles must have `rc` and `rc_addr` set.

### 2.2 Start/Stop Mount from App

**Problem:** StartMount exists but unmount is not available via the UI.

**Action:** Add unmount via RC API (`mount/unmount` endpoint). Show mount/unmount toggle button per profile. Requires RC enabled.

### 2.3 Unsaved Changes Warning

**Problem:** No warning when navigating away or closing the app with unsaved profile changes.

**Action:** Track dirty state per profile. Warn on navigate/close with "Save / Discard / Cancel" dialog.

### 2.4 TOML Config + Script Metadata (Hybrid Architecture)

**Problem:** Config is stored as profiles.json. User wants TOML. Also wants scripts to be self-describing for re-import.

**Action:** Migrate config storage from JSON to TOML. Generated scripts include a metadata header block (TOML in comments) that the app can re-import. Config file is the source of truth; scripts are output with embedded metadata for recovery.

**Design doc needed** before implementation.

## Phase 3: ViewModel Refactoring (Medium Priority)

### 3.1 Split MainWindowViewModel

**Problem:** MainWindowViewModel is ~1024 lines handling profiles, backend discovery, mount operations, script generation, logging, and UI state.

**Action:** Split into 5-6 focused ViewModels:

| ViewModel | Responsibility | Est. size |
|-----------|---------------|-----------|
| MainWindowViewModel | Window state, theme, navigation | ~100 lines |
| ProfileListViewModel | Profile CRUD, selection, persistence | ~150 lines |
| ProfileDetailViewModel | Edit selected profile, save/mount | ~250 lines |
| MountOptionsViewModel | Already exists — typed parameter editor | ~175 lines |
| BackendOptionsViewModel | Backend discovery, remote creation | ~150 lines |
| ActivityLogViewModel | Log entries, status bar | ~50 lines |

Communication via direct property binding and event callbacks.

**Do this before adding major new features** to prevent MainWindowViewModel from growing further.

## Phase 4: Distribution (High Priority)

### 4.1 macOS .app Bundle

**Action:** Create `scripts/build-macos-app.sh` that:
- Runs `dotnet publish -r osx-arm64 --self-contained -c Release`
- Creates `RcloneMountManager.app` with proper `Info.plist`
- Copies icon

### 4.2 DMG Creation

**Action:** Create `scripts/create-dmg.sh` that:
- Creates `.dmg` with app + Applications symlink
- Background image, icon positioning
- SHA256 checksum generation

### 4.3 Code Signing

**Action:** Add code signing placeholder in build script for Apple Developer ID. Initially unsigned (Gatekeeper bypass required).

### 4.4 CI/CD via GitHub Actions

**Action:** Create `.github/workflows/release-dmg.yml`:
- Trigger on tag push (`v*`)
- Matrix build: arm64 + x64
- Upload DMG as release asset with checksums
