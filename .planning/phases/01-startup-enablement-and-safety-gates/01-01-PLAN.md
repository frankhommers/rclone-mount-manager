  <action>Implement `StartupPreflightService.RunAsync(MountProfile profile, CancellationToken cancellationToken)` that executes checks in order: (1) rclone binary resolvable/executable, (2) mount path resolves and is writable/creatable, (3) cache path resolves and is writable when cache-related options are configured, (4) credentials are available for selected quick-connect mode. Classify failing checks per research guidance (critical vs warning), and return a report with explicit per-check messages. Reuse existing parsing/path conventions from core services; do not mutate profile state in this service.</action>
  <verify>dotnet test RcloneMountManager.slnx --filter "ClassName=StartupPreflightServiceTests"</verify>
  <done>Service returns a full typed report where any critical failure flips `CriticalChecksPassed` to false and includes explicit reasons.</done>
